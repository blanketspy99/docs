class mycommon.filesystem.spfs.SharePointFileSystem(username, client_id, tenant_id, scope=[], hostname='', documentRoot='Documents', sitePath=None, password=None, passwordEnv=None, cacheToken=True, cache_style=<class 'mycommon.filesystem.spfs.LocalSharePointCredsCache'>, with_auth=True)#

Bases: FileSystem

Initialize sharepoint filesystem as normal filesystem. Note: Sharepoint requires authentication, and, current module supports 2 types 1. Username and Password 2. Interactive using device code flow. This method is generally required when user login is backed with MFA.

Parameters

  • username (str) – Sharepoint username, this will be used for authentication/cache refresh

  • client_id (str) – Azure Active Directory Application Client Id

  • tenant_id (str) – Azure Active Directory Application Client Id

  • scope (list, optional) – Active Directory Application access scopes, by default []

  • hostname (str, optional) – Sharepoint hostname, by default “”

  • documentRoot (str, optional) – Document base folder name, Ex: “Document Library”, by default ‘Documents’

  • sitePath (str, optional) – Sharepoint site path. Ex: /sites/myanalytics, by default None

  • password (str, optional) – Password for the username, required only if authentication is done by username and password, by default None

  • passwordEnv (str, optional) – Environment variable which contains the password. passwordEnv takes precedence over the password parameter, by default None

  • cacheToken (bool, optional) – Do you want to enable cache, helps to reduce the use of credentials everytime user logs in, by default True

  • cache_style (Callable[[str],Union[dict,None]], optional) – Method to fetch/write into cache using given username. LocalSharePointCredsCache is created for you which will use user home directory to store credential which can be overridden using SHAREPOINT_CREDENTIALS_CACHE_FILE environment variable. Note: method should take username as parameter and implement read & write function for cache. , by default LocalSharePointCredsCache

  • with_auth (bool, optional) – True if you want to initiate the instance with login else you may want to use authenticate later for login , by default True

Raises

Exception –

write_file_content(path, data, show_progress=True, chunk_write=True, chunk_size=10485760, content_range=None, **kwargs)#

Use this method to write binary data to Sharepoint filepath

Parameters

  • path (str) – Location of path(including filename) where data needs to be written

  • data (file like object) – binary buffer data that needs to be written into file should implement read method

  • show_progress (bool, optional) – To show progress in write mode. it is set False in append mode., by default True

  • chunk_write (bool, optional) – write data into file in chunks., by default True

  • chunk_size (int, optional) – amount of chunk that needs to write in each iteration. This parameter is meaningful when chunk_write=True, by default 4**10*10 (10MB)

  • content_range (Optional[str], optional) – requires bytes content range objects. Ex: “bytes 0-499/500”, by default None

Return type

dict

Returns

  • dict – object file head metadata (get_file_head)

  • Note (If chunk_write is True and content_range is None, then in order to write in chunks, module needs)

  • input data content length in bytes, for which you need to pass addition kwargs containing ContentLength in bytes.

  • Example (**{‘ContentLength’:1534746734653})
